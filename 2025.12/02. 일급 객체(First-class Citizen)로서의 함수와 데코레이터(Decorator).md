## 0) 일급 객체(First-class object) 정의

어떤 언어에서 어떤 대상이 **일급 객체**라면, 그 대상은 “값(value)”처럼 완전한 시민권을 가져서 보통 아래가 가능:

1. **변수에 담을 수 있다**
    
2. **함수의 인자로 전달할 수 있다**
    
3. **함수의 반환값으로 돌려줄 수 있다**
    
4. **자료구조(리스트/딕트 등)에 저장할 수 있다**
    
5. (많은 설명에서 덧붙여) **런타임에 생성될 수 있다**
    

파이썬의 함수는 이걸 다 만족해서 **함수는 일급 객체**.

---

## 1) “함수도 객체다” 10초 증명 (type / id / callable)

```python
def add(a, b):
    return a + b

print(type(add))      # <class 'function'>  -> 함수도 타입(클래스)을 가진 객체
print(id(add))        # 메모리 상의 정체성(객체 식별자)
print(callable(add))  # True               -> 호출 가능한 객체
```

발표 멘트 예시:

- “`add`는 실행 결과가 아니라 **함수 객체 자체**입니다.”
    
- “`type`이 function이고, `id`가 있고, `callable`이 True니까 **함수는 객체(값)**로 존재합니다.”
    

---

## 2) 5가지 권리(=일급의 근거) 예시

### (1) 변수를 담을 수 있음

```python
f = add       # 함수를 변수에 저장 (호출 X)
print(f(2, 3))  # 5
```

포인트: `f = add()`가 아니라 `f = add` 임.

---

### (2) 함수 인자로 전달됨 (콜백)

```python
def apply(fn, x, y):
    return fn(x, y)

print(apply(add, 10, 20))  # 30
```

포인트: “행동(로직)을 데이터처럼 전달한다.”

---

### (3) 함수가 리턴됨 (함수 생성기)

```python
def make_multiplier(k):
    def mul(x):
        return x * k
    return mul

times3 = make_multiplier(3)
print(times3(7))  # 21
```

포인트: 함수가 함수를 “만들어서” 반환한다 → 여기서 클로저로 이어질 준비가 됨.

---

### (4) 자료구조에 들어감 (전략 선택)

```python
def inc(x): return x + 1
def dec(x): return x - 1

ops = {"up": inc, "down": dec}
print(ops)    # 11
print(ops)  # 9
```

포인트: “조건문(if/elif) 대신 딕트로 동작을 선택” 같은 패턴.

---

### (5) 런타임에 생성될 수 있음 (중첩 함수)

```python
def outer():
    def inner():
        return "I am created at runtime!"
    return inner

fn = outer()
print(fn())  # I am created at runtime!
```

포인트: 함수 정의가 “코드에 적혀있어도”, 실제로 **함수 객체는 실행 중에 만들어질 수 있다**.
