# 옵시디언 용으로 만들어서 css 적용 안되있음!

> [!m-callout] GIT
> Git은 **분산 버전 관리 소프트웨어 시스템**으로, 소스 코드나 데이터의 버전을 관리할 수 있다. 주로 여러 프로그래머가 협업하여 소프트웨어를 개발할 때 소스 코드를 관리하는 데 사용된다.
>
> - **원 개발자**: 리누스 토르발스
> - **개발자**: 주니오 하마노 외
> - **최초 공개**: 2005년 4월 7일 (20년 전)
> - **안정 버전**: 2.52.0 (2025년 11월 17일)
> - **저장소**: git.kernel.org/pub/scm/git/git.git
> - **작성 언어**: 주로 C, GUI 및 스크립트는 셸 스크립트, Perl, Tcl, Python
> - **운영체제**: POSIX (Linux, macOS, Solaris, AIX), Windows
> - **사용 언어**: 영어
> - **유형**: 버전 관리 시스템
> - **라이선스**: GPL-2.0-only
> - **웹사이트**: git-scm.com

위는 GIT에 대한 위키피디아에 정의이다. 분산 / 버전 / 소프트웨어 / 시스템 / 소스 코드 / 데이터 의 의미를 전부 알아야 이해할 수 있을것이다.

> [!info|label] 분산
>
> - 분산은 말 그대로 한곳에 모이지 않고 여러 곳으로 나뉘어 퍼져 있음을 뜻한다.
> - Git에서의 분산은 단순한 “여러 개”가 아니라, **모든 참여자가 ‘완전한 저장소’를 각각 가지고 있다는 의미**
> - 누구나 전체 역사가 담긴 원본을 가질 수 있다

> [!info|label] 버전(version)
>
> - 분산은 말 그대로 한곳에 모이지 않고 여러 곳으로 나뉘어 퍼져 있음을 뜻한다.
> - Git에서의 분산은 단순한 “여러 개”가 아니라, **모든 참여자가 ‘완전한 저장소’를 각각 가지고 있다는 의미** > **소프트웨어**란

> [!info|label] 소프트 웨어
>
> - 소프트웨어란 컴퓨터가 무엇을, 어떻게 할지 지시하는 논리와 명령의 집합.
> - 눈에 보이고 만질 수 있는 하드웨어와 달리, 형태는 없지만 동작을 결정하는 ‘규칙과 설계’.
> - 기계를 움직이게 하는 생각과 절차를 코드로 표현한 것.
> - soft인 이유는 수정 가능하고, 복사 가능하기 때문! (hard와 반대)

> [!m-callout] 결국 GIT 이란?
> Git은 모든 개발자가 동일한 전체 이력(history)을 가진 저장소를 각자 보유하는 분산 버전 관리 시스템이다
